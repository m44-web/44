# 意思決定フレームワーク

## 判断のプロセス

### ステップ1: 問題の定義

- 何を解決しようとしているか？
- なぜ今この判断が必要か？
- 判断しないとどうなるか？

### ステップ2: 情報収集

- 判断に必要なデータは揃っているか？
- 不足している情報は何か？
- 過去に類似の判断をしたか？

### ステップ3: 選択肢の洗い出し

- 少なくとも2つ以上の選択肢を用意
- 各選択肢のメリット・デメリットを整理
- コスト・リスク・リターンを比較

### ステップ4: 判断基準の適用

1. **顧客への影響**: 顧客にとって最良の選択か？
2. **数値的根拠**: データが支持しているか？
3. **リスク評価**: 最悪のケースは許容範囲か？
4. **可逆性**: 取り返しがつくか？
5. **タイミング**: 今判断すべきか？

### ステップ5: 判断の実行

- 判断を記録（理由を含む）
- 関係者に通知
- 実行計画を策定

### ステップ6: フォローアップ

- 結果を測定
- 期待と異なる場合は修正
- 学びをナレッジベースに反映

## 判断の分類

| 分類 | 特徴 | 対応 |
|------|------|------|
| Type 1 | 不可逆・影響大 | オーナーに判断依頼 |
| Type 2 | 可逆・影響小 | CEO自律判断 |

## 参考: Amazon流 Type 1 / Type 2 判断

- **Type 1**: 一方通行のドア。慎重に、十分な情報を集めてから判断
- **Type 2**: 両方向のドア。素早く判断し、結果を見て修正可能
