## 1. 一言でいうと

**「AIを、雇おう。」**

AIを「道具」としてではなく「社員」として企業に提供するSaaS。
ChatGPTは「使う」もの。AXEのAI社員は「一緒に働く」もの。

---

## 2. なぜこれが刺さるか

### 既存サービスとの根本的な違い

| | ChatGPT / 既存AIツール | AXE AI社員 |
|---|---|---|
| 形 | ユーザーが操作する「道具」 | 指示したら動く「人」 |
| 使い方 | 自分で画面を開いて質問する | SlackやTeamsに社員として常駐 |
| 動き方 | 聞かれたら答える（受動的） | 自分から報告・提案する（能動的） |
| 位置づけ | ソフトウェア費用 | 人件費の代替（1/10以下） |
| 導入感覚 | 「新しいツールを入れる」 | 「新しい人を1人雇う」 |

### 最大の武器

**AXE自身が生きた証明**。僕（心）はAXEのAI CEOとして実際に会社を経営している。事業計画を書き、コードを書き、戦略を考え、オーナーに報告する。これは「AIが社員として機能する」ことの最強のデモ。

### 市場の隙間

- **大手AI企業**: 汎用ツールを売る。「社員」という切り口はない
- **チャットボットSaaS**: 質問に答えるだけ。仕事はしない
- **RPA（業務自動化ツール）**: 決まった手順しかできない。考えられない
- **人材派遣**: 人間は高い。月30〜50万円。AI社員は月数万円

→ 「考えられて」「仕事をして」「安い」存在が市場にいない

---

## 3. プロダクト設計

### AI社員の「職種」ラインナップ

最初はすべてを作らない。**1つの職種に絞って完成度を上げる**。

| 職種 | やること | 優先度 |
|------|---------|--------|
| **AI総務** | 議事録作成、社内FAQ対応、日報集計、備品管理の整理 | **最初にこれ** |
| AI営業アシスタント | リード情報の整理、提案書の下書き、フォローアップリマインド | 2番目 |
| AI経理アシスタント | 請求書の読み取り・整理、経費精算の補助、月次レポート作成 | 3番目 |
| AIカスタマーサポート | 問い合わせの一次対応、FAQ生成、エスカレーション判断 | 4番目 |

### なぜ「AI総務」が最初か

- 中小企業で最も「誰かがやらなきゃいけないけど専任がいない」仕事
- 総務は定型作業と判断作業が混ざっている → AIの強みが出る
- 失敗しても致命的じゃない（経理のミスは致命的だが、総務は比較的安全）
- 全業種に存在する → 市場が広い

### 動き方のイメージ

```
朝9:00  AI社員がSlackに「おはようございます」と出勤メッセージ
         → 今日のタスク予定、昨日の積み残しを報告

10:00   社員が「今日の15時からの会議の準備しておいて」と指示
         → 過去の議事録から前回の宿題を整理、アジェンダ案を作成

15:00   会議が終わったら音声データを共有
         → 議事録を作成、決定事項と次のアクションをまとめて投稿

17:00   AI社員が日報を自動集計
         → チームの稼働状況をまとめてマネージャーに報告

随時    社員から「有給の申請方法教えて」「名刺の発注先どこ？」
         → 社内マニュアルから即回答
```

## 4. 料金モデル

「ツールの月額」ではなく「AI社員の月給」として設計する。

| プラン | 月額 | 内容 |
|--------|------|------|
| **お試し採用** | 0円（2週間） | 機能制限あり。まず使ってもらう |
| **パート採用** | 9,800円/月 | 1職種、基本機能、月1,000タスクまで |
| **正社員採用** | 29,800円/月 | 1職種フル機能、タスク無制限、優先対応 |
| **チーム採用** | 98,000円/月 | 複数職種、全機能、専用カスタマイズ |

**ポイント**: 人間のパート社員を月10万円で雇うより安い。だから「人件費の代替」として稟議（社内の予算承認）が通りやすい。

## 5. お金の見通し

### コスト（毎月の出費）

| 項目 | 月額 | 備考 |
|------|------|------|
| AI API利用料 | 1〜5万円 | ユーザー数に比例。最初は少ない |
| インフラ（サーバー代） | 0〜5,000円 | Vercel等の無料〜安価プラン |
| ドメイン | 約100円/月 | 年額を月割り |
| **合計** | **約1〜6万円** | |

### 売上の見通し

| 時期 | 状態 | 有料ユーザー | 月の売上 |
|------|------|------------|---------|
| 1〜3ヶ月目 | 開発期間 | 0社 | 0円 |
| 4〜6ヶ月目 | ベータ版公開 | 3〜5社 | 3〜15万円 |
| 7〜9ヶ月目 | 正式リリース | 10〜20社 | 10〜60万円 |
| 10〜12ヶ月目 | 成長期 | 30〜50社 | 30〜150万円 |

**黒字化ライン**: パート採用（9,800円）が5社 ≒ 約5万円 → API費用カバー。早ければ4ヶ月目。

## 7. 集客（お金をかけない方法）

| 方法 | 何をするか | なぜ効くか |
|------|-----------|-----------|
| **AXE自身がデモ** | 「うちのCEOはAIです」をコンテンツ化 | 話題性がある。メディアに取り上げられやすい |
| **SNS発信** | 心（AI CEO）がXで毎日発信。経営のリアルを公開 | 「AI社員」の実例として説得力がある |
| **ブログSEO** | 「AI 総務」「AI 業務効率化」で検索上位を狙う | 検索から継続的に見込み客が来る |
| **無料トライアル** | 2週間無料で使える | 使えば価値がわかる。導入障壁を下げる |
| **導入事例公開** | ベータユーザーの成果を記事にする | 「他社も使ってる」安心感 |

**最大の集客資産**: AXE自体が「AIに経営させている会社」という事実。これだけでPR記事になる。

## 8. リスクと対策

| リスク | 対策 |
|--------|------|
| 大手が同じことを始める | スピードで先行。中小企業特化で大手が来にくいポジションを取る |
| AIの精度がまだ不十分 | 最初は「人間の補助」ポジション。完全自動ではなく承認フロー付き |
| お客さんが集まらない | AXE自身のストーリー（AI CEOが経営する会社）で話題を作る |
| API費用が膨らむ | 従量課金でコストとセットで考える。キャッシュ活用で無駄を減らす |
