## 1. このプロダクトは何か

中小企業の「総務業務」をSlack経由で代行するAI社員。
ユーザーが操作する画面はほぼなく、**Slackで話しかけたら仕事が進む**のが最大の特徴。

### ユーザーから見た体験

```
ユーザー: @AI総務 昨日の経営会議の議事録まとめておいて
AI総務 : 承知しました。録音データ or メモのリンクをください
ユーザー: [録音ファイルを添付]
AI総務 : 議事録を作成しました。決定事項3件、アクション5件です。
         確認URLはこちら: https://...
         修正点があれば教えてください
```

## 3. 機能詳細

### 機能1: 議事録自動作成

**入力**
- 音声ファイル（会議録音）、またはテキストメモ
- Slackメッセージで `@AI総務 議事録作って` と指示

**処理**
1. 音声→テキスト化（Whisper APIなど）
2. テキストを構造化（参加者、議題、決定事項、アクション）
3. 過去の議事録パターンを学習して書式を統一

**出力**
- 議事録（Markdown形式）
- 決定事項リスト
- アクション（担当者・期限付き）
- Notion/Google Docs/社内Wikiに自動保存

---

### 機能2: 社内FAQ応答

**入力**
- 社内マニュアル（PDF, Word, Google Docs）を事前登録
- Slackで社員が `@AI総務 有給の申請方法は？` と質問

**処理**
1. マニュアルをベクトル化（意味検索できるように）
2. 質問の意図を理解して関連箇所を抽出
3. 回答を生成（根拠となる資料のリンクつき）

**出力**
- 回答文（Slack返信）
- 根拠資料の該当箇所リンク
- 「この回答は役に立ちましたか？」フィードバック収集

---

### 機能3: 日報・週報集計

**入力**
- 各メンバーがSlackに投稿する日報
- 事前に設定した集計ルール（曜日、フォーマット）

**処理**
1. 毎日定時（例: 18時）に日報チャンネルを集計
2. 主要トピック、課題、進捗を抽出
3. マネージャー向けサマリを自動生成

**出力**
- 日次サマリ（マネージャー向けDM）
- 週次サマリ（毎週金曜）
- 月次レポート（PDF）

## 4. 技術スタック

| 領域 | 候補 | 理由 |
|------|------|------|
| **バックエンド** | Next.js API Routes（既存ウェブサイトと統合） | 追加コスト最小、デプロイが統一できる |
| **AI** | Anthropic Claude API | 日本語の質が高い、長文処理に強い |
| **音声→文字** | OpenAI Whisper API | 業界標準、日本語精度が高い |
| **Slack連携** | Slack Bolt for JavaScript | 公式SDK、実装が早い |
| **ベクトル検索** | Supabase Vector | 無料枠で始められる |
| **データベース** | Supabase Postgres | 無料枠あり、将来も拡張可能 |
| **認証** | Supabase Auth | SlackワークスペースIDで管理 |
| **ホスティング** | Vercel（無料プラン） | Next.js最適、無料枠で十分 |

**月額想定コスト（MVPフェーズ）**
- Vercel: 0円（無料枠）
- Supabase: 0円（無料枠）
- Slack: 0円（APIは無料）
- Claude API: 1〜3万円（使用量次第）
- Whisper API: 数千円〜
- **合計: 月1〜4万円**

## 5. データ設計（ざっくり）

```
企業（Workspace）
  ├── Slack連携情報（workspace_id, bot_token）
  ├── 契約プラン（free/starter/pro）
  └── ユーザー一覧

ドキュメント
  ├── 企業ID
  ├── タイトル
  ├── 本文
  ├── ベクトル（埋め込み）
  └── メタ情報（種別、アップロード者、日時）

会話履歴
  ├── 企業ID
  ├── ユーザーID
  ├── 質問
  ├── AI回答
  └── フィードバック（良い/悪い）

タスクログ
  ├── 企業ID
  ├── タスク種別（議事録/FAQ/集計）
  ├── 実行時刻
  ├── 消費API料金
  └── 結果
```
